TROUBLESHOOTING
===============

AWG не подключается у некоторых мобильных операторов, а WG работает
-------------------------------------------------------------------
Основные отличия AWG от обычного WG:
- AWG добавляет obfuscation параметры;
- handshake и первые UDP-пакеты могут иметь другой размер и профиль;
- агрессивные Jc/Jmin/Jmax могут хуже проходить через некоторые мобильные сети;
- MTU нужно проверять и на сервере, и на клиенте.

Первый безопасный профиль для проверки
--------------------------------------
Создайте отдельный тестовый интерфейс, например awg1:

  server MTU = 1280
  client MTU = 1280
  ListenPort = 443 или другой реально открытый UDP-порт
  Jc = 4..7
  Jmin = 10..50
  Jmax = 200..1000

Если хотите проверить меньший client MTU:

  server MTU = 1280
  client MTU = 1200

Для IPv6 не опускайте MTU ниже 1280 без отдельного понимания последствий.

Как проверить, виноват ли порт/оператор
---------------------------------------
На сервере во время попытки подключения:

  sudo tcpdump -ni any udp port 56789

или если интерфейс слушает 443:

  sudo tcpdump -ni any udp port 443

Интерпретация:

1. Пакетов нет вообще
   Скорее всего, оператор/маршрут/провайдерский firewall/порт не пропускает UDP до сервера.

2. Пакеты есть, но handshake не появляется
   Проверяйте совпадение AWG параметров, ключи, endpoint, MTU, AllowedIPs, серверный firewall.

3. Handshake есть, но интернет не работает
   Проверяйте nftables masquerade, ip_forward, AllowedIPs клиента и DNS.

Проверка состояния интерфейсов
------------------------------

  sudo ./install.sh --status

или:

  sudo ./scripts/00_manage.sh --status

Проверка сервиса:

  sudo systemctl status awg-quick@awg0.service
  sudo journalctl -u awg-quick@awg0.service -n 100 --no-pager

Проверка AWG peers:

  sudo awg show

Проверка маршрутизации и firewall:

  sudo nft list ruleset
  cat /etc/sysctl.d/00-amnezia.conf
  sysctl net.ipv4.ip_forward

Почему лучше добавлять второй интерфейс для теста
------------------------------------------------
Если текущий awg0 работает у части клиентов, не меняйте его резко. Создайте awg1 с другим портом, другой подсетью, другим MTU и более мягкими AWG параметрами. Так вы сможете тестировать проблемного оператора без риска сломать рабочий awg0.

Пример:

  sudo ./install.sh
  -> 3) Создать новый интерфейс AWG

Параметры:

  iface: awg1
  port: 443
  IPv4: 10.8.2.1/24
  IPv6: fd42:42:43::1/64
  server MTU: 1280
  client MTU: 1280
  Jc: 4
  Jmin: 10
  Jmax: 200

После этого добавьте клиента именно на awg1 и проверьте его у проблемного оператора.

Если порт 443 пробрасывается на 56789
-------------------------------------
Лучше проверить отдельный вариант без DNAT/REDIRECT:

  ListenPort = 443

на самом AWG-интерфейсе. Так исключается дополнительный слой NAT и проще понять, что реально видит серверный tcpdump.

IPv6 и AllowedIPs
-----------------
По умолчанию bundle создаёт IPv4-only клиента:

  AllowedIPs = 0.0.0.0/0

Это безопаснее, если серверный IPv6 не настроен полностью. Если включить ::/0 без рабочего IPv6 firewall/route/NAT, часть клиентов может вести себя нестабильно.

Для включения IPv6:

  sudo CLIENT_ENABLE_IPV6=yes ./scripts/04_add_client.sh phone awg0

Но сначала убедитесь, что IPv6 на сервере действительно настроен.
