CONFIG_GUIDE
============

Файлы состояния
---------------

  /etc/amnezia/amneziawg/install.env
    Пути к awg, awg-quick, cache/source и sysctl. Перезаписывается атомарно с backup.

  /etc/amnezia/amneziawg/manager.env
    Указатель на последний/активный интерфейс для совместимости. Перезаписывается атомарно с backup.

  /etc/amnezia/amneziawg/manager-awg0.env
  /etc/amnezia/amneziawg/manager-awg1.env
    Настройки конкретного интерфейса: SERVER_CONF, CLIENTS_DIR, KEYS_DIR, ENDPOINT_HOST, ENDPOINT_PORT, DEFAULT_CLIENT_MTU, SERVER_ENABLE_IPV6.

  /etc/amnezia/amneziawg/firewall.env
    Последние EXTERNAL_IF, SSH_PORT, список AWG_PORTS и AWG_IFACES. Нужен для чистки старых nftables-правил при удалении интерфейса.

  /etc/amnezia/amneziawg/awg0.conf
  /etc/amnezia/amneziawg/awg1.conf
    Серверные конфиги интерфейсов.

  /etc/amnezia/amneziawg/clients/
  /etc/amnezia/amneziawg/keys/
    Совместимый layout для awg0.

  /etc/amnezia/amneziawg/awg1/clients/
  /etc/amnezia/amneziawg/awg1/keys/
    Layout для второго и последующих интерфейсов.

Backup layout
-------------
Каждая операция создаёт папку вида:

  /etc/amnezia/amneziawg/backups/YYYYMMDD-HHMMSS-label/

Внутри:

  INFO
    label и время создания.

  MANIFEST.tsv
    Список старых файлов/директорий, которые были сохранены перед изменением.

  CREATED_PATHS
    Список новых путей, которые нужно удалить при rollback/restore. Есть не во всех backup.

  files/...
    Копии старых файлов с сохранением абсолютного пути внутри backup.

Откат:

  sudo ./scripts/10_restore_backup.sh /etc/amnezia/amneziawg/backups/YYYYMMDD-HHMMSS-label

Перед restore создаётся отдельный pre-restore backup текущего состояния.

MTU и IPv6
----------
IPv6 требует MTU не ниже 1280. Поэтому:

- при создании серверного интерфейса с IPv6 server MTU < 1280 запрещён;
- при добавлении клиента с CLIENT_ENABLE_IPV6=yes client MTU < 1280 запрещён;
- по умолчанию новые клиенты IPv4-only, чтобы awg-quick не добавлял IPv6 /128 route.

Рекомендуемые стартовые значения:

  SERVER_MTU_DEFAULT=1280
  CLIENT_MTU_DEFAULT=1280

Для IPv4-only тестов в сложных мобильных сетях можно пробовать client MTU 1200, но не включайте для такого клиента IPv6.

Obfuscation параметры
---------------------
Jc:
  Количество junk-пакетов/итераций обфускации. Больше значение = больше шума и накладных расходов.

Jmin:
  Минимальный размер junk-пакета.

Jmax:
  Максимальный размер junk-пакета. Должен быть >= Jmin.

S1/S2:
  Размеры служебных obfuscation-пакетов/сигнатур. Обычно оставляют defaults.

H1-H4:
  32-bit magic/header значения. Клиент и сервер должны иметь одинаковые значения.

Если нет конкретной причины менять параметры, оставляйте defaults.

Подсети по умолчанию
--------------------
Если уже существует:

  awg0: 10.8.1.1/24

следующий интерфейс получит default:

  awg1: 10.8.2.1/24

IPv6 default считается синхронно:

  10.8.1.1/24 -> fd42:42:42::1/64
  10.8.2.1/24 -> fd42:42:43::1/64
  10.8.3.1/24 -> fd42:42:44::1/64

nftables модель
---------------
03_setup_nftables.sh строит итоговый /etc/nftables.conf для всех AWG интерфейсов. Файл начинается с:

  flush ruleset

Это сделано, чтобы при `nft -f /etc/nftables.conf` runtime ruleset заменялся целиком, а старые правила не наслаивались поверх новых.

Для нескольких интерфейсов важные строки выглядят так:

  iifname "ens3" udp dport { 56789, 520 } accept
  iifname { "awg0", "awg1" } oifname "ens3" masquerade

При удалении интерфейса скрипт использует firewall.env и параметры EXTRA_AWG_PORTS_TO_REMOVE / EXTRA_AWG_IFACES_TO_REMOVE, чтобы убрать старые allow/NAT правила.

Неинтерактивные примеры
-----------------------
Создать awg1:

  sudo VPN_IF_DEFAULT=awg1 \
       LISTEN_PORT_DEFAULT=520 \
       SERVER_ADDR_V4_DEFAULT=10.8.2.1/24 \
       SERVER_ADDR_V6_DEFAULT=fd42:42:43::1/64 \
       ENDPOINT_HOST_DEFAULT=203.0.113.10 \
       SERVER_MTU_DEFAULT=1280 \
       CLIENT_MTU_DEFAULT=1280 \
       ./scripts/02_create_server_config.sh

Добавить IPv4-only клиента:

  sudo CLIENT_MTU=1280 QR_OUTPUT=no ./scripts/04_add_client.sh phone awg1

Добавить IPv6-клиента:

  sudo CLIENT_ENABLE_IPV6=yes CLIENT_MTU=1280 ./scripts/04_add_client.sh phone6 awg1

Удалить клиента:

  sudo ./scripts/08_remove_client.sh phone awg1

Удалить интерфейс без немедленного применения nftables:

  sudo CONFIRM_REMOVE=yes NFT_SAVE_CHANGES=yes NFT_APPLY_NOW=no ./scripts/09_remove_interface.sh awg1

Восстановить backup без применения firewall и без restart:

  sudo RESTORE_CONFIRM=yes RESTORE_APPLY_NFT=no RESTORE_RESTART_SERVICES=no \
       ./scripts/10_restore_backup.sh /etc/amnezia/amneziawg/backups/YYYYMMDD-HHMMSS-label
