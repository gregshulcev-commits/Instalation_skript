CHANGES_AND_AUDIT
=================

Ключевые найденные проблемы
---------------------------

1. Новые клиенты автоматически получали IPv6 /128 в server.conf AllowedIPs.
   Это заставляло awg-quick выполнять `ip -6 route add ... dev awgN`. В сочетании с MTU 800 и/или неполной IPv6-схемой интерфейс мог падать при поднятии.

   Исправлено:
   - по умолчанию server peer получает только IPv4 /32;
   - IPv6 /128 добавляется только при CLIENT_ENABLE_IPV6=yes;
   - IPv6 + MTU < 1280 запрещён.

2. nftables обновлялся дозаписью, из-за чего появлялись дубли базовых input-правил, UDP allow и NAT masquerade.

   Исправлено:
   - 03_setup_nftables.sh строит итоговый candidate и перезаписывает nftables.conf с backup;
   - итоговый файл начинается с flush ruleset;
   - AWG UDP ports и AWG masquerade объединяются;
   - старая table ip amneziawg_bundle удаляется из candidate;
   - пользовательские redirect/DNAT/Grafana rules сохраняются при понятной структуре firewall.

3. manager/install/firewall env файлы раньше могли разрастаться append-блоками.

   Исправлено:
   - install.env, manager.env, manager-IFACE.env, firewall.env перезаписываются атомарно;
   - перед перезаписью создаётся timestamp backup.

4. Не было полноценного rollback/restore механизма.

   Исправлено:
   - добавлены transaction helpers в 00_common.sh;
   - операции создания/добавления/удаления создают backup с MANIFEST.tsv;
   - Ctrl+C/ошибка удаляет новые файлы и восстанавливает старые;
   - добавлен 10_restore_backup.sh.

5. Не было удаления клиента и интерфейса.

   Исправлено:
   - добавлен 08_remove_client.sh;
   - добавлен 09_remove_interface.sh;
   - пункты добавлены в 00_manage.sh.

6. Defaults были неудобны.

   Исправлено:
   - следующий IPv4 default выбирается по существующим *.conf;
   - IPv6 default синхронизирован с IPv4;
   - endpoint host default берётся из `ip -4 route get 1.1.1.1` source address;
   - добавлены подсказки по obfuscation-параметрам.

7. 05_update_amneziawg.sh работал только с одним текущим интерфейсом.

   Исправлено:
   - update сканирует все *.conf и обрабатывает все awg-quick@IFACE.service.

8. Тесты не покрывали важные сценарии.

   Исправлено тестами:
   - bash -n всех скриптов;
   - fake install;
   - создание awg0/awg1;
   - nftables template;
   - обновление существующего native nftables без дублей;
   - IPv4-only клиент по умолчанию;
   - явный IPv6-клиент;
   - update install.env с backup;
   - 00_manage --status;
   - удаление клиента и restore;
   - удаление интерфейса, очистка nftables и restore.

Ограничения
-----------

1. Реальное поднятие интерфейса на вашем VPS не выполнялось в контейнере: нет вашего ядра, systemd runtime, DKMS окружения и реального amneziawg kernel module.

2. Автовыдача IPv4 рассчитана на /24.

3. Автоматическое редактирование nftables рассчитано на понятный native ruleset. Для сложного firewall скрипт остановится и покажет ручные инструкции.
