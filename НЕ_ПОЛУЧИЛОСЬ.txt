Что не получилось полностью закрыть в этой сборке
================================================

1. В архив не вложены реальные upstream tar.gz исходников AmneziaWG.
   Bundle умеет скачать исходники сам при наличии интернета. Для офлайн-установки используйте:

     sources/download_upstream_sources.sh

   После скачивания архивов папку sources/ можно перенести на целевой сервер.

2. Полная реальная сборка DKMS модуля против настоящего ядра VPS не выполнялась в контейнере.
   Причина: контейнер не является целевым сервером с его ядром, headers, DKMS runtime, Secure Boot и systemd.

3. Вместо этого выполнены сухие тесты на заглушках:
   - bash -n всех скриптов;
   - установка state/install.env;
   - создание awg0;
   - server MTU и DEFAULT_CLIENT_MTU;
   - nftables для одного интерфейса;
   - добавление клиента;
   - создание awg1 без поломки awg0;
   - nftables для двух интерфейсов;
   - добавление клиента именно в awg1;
   - проверка единого мастера 00_manage.sh --status.

4. nftables всё ещё генерируется как полный ruleset с flush ruleset.
   Это удобно для чистого VPS, но если на сервере уже есть сложные firewall-правила, их нужно вручную объединить с правилами из scripts/03_setup_nftables.sh.

5. Автовыдача клиентских IPv4 адресов рассчитана на /24 подсети.
   Поэтому скрипт создания интерфейса валидирует IPv4 адрес сервера как 10.8.X.1/24.
